4. Key Technical Considerations
Immersive 3D Background (The Galaxy)
Implement a global Canvas via @react-three/fiber that persists across route changes.
The background must render a realistic nebula using custom shaders or particle systems (PointsMaterial).
Performance: Use instancedMesh for stars to handle thousands of particles without frame drops.
Responsive Design & Adaptive 3D
Desktop: Full interactive 3D galaxy with mouse-move parallax.
Mobile: Reduce particle count. The central "Sun" menu must fallback to a high-performance accessible Burger Menu if WebGL performance is low.
Breakpoints:
sm: 640px (Mobile - Simplified effects)
md: 768px (Tablet - Full layout)
lg: 1024px (Desktop - High fidelity)
State Management
Use Zustand (lightweight store) to manage the "Galaxy State" (speed, rotation, zoom level) and "Menu State" (Sun expansion).
Avoid Prop Drilling.
Use React Context only for Theme/Toast notifications.
Animation Implementation
3D: Use useFrame (R3F) for the constant rotation of the nebula and the "Sun" pulsation.
UI: Use Framer Motion for DOM elements.
Scroll-triggered animations (fade-in-up) for the content sections (History, CV).
Smooth transitions between "Passive Galaxy" and "Active Menu" states.
5. Development Best Practices
Code Organization & Type Safety
Every component must be typed (interface Props).
Follow the Container/Presentational pattern where possible (Server Components fetch data, Client Components render UI).
Keep 3D logic separate from HTML UI logic.
Styling Methodology (Nebula Theme)
Use Tailwind utility classes for layout and typography.
Z-Index Strategy:
z-0: Canvas (Galaxy Background).
z-10: Content/Scrollable Sections.
z-50: Navigation/Sun Menu.
Glassmorphism: Use backdrop-blur and semi-transparent backgrounds (bg-black/30) to ensure text is readable over the starry background.
Performance Considerations
Three.js: Dispose of geometries/materials when unmounting. Use drei/PerformanceMonitor to lower quality on weak devices.
Lazy Loading: Use next/dynamic to load heavy 3D components only after the main UI is interactive.
Fonts: Use next/font with a professional sans-serif (e.g., Inter or Space Grotesk) to prevent Layout Shift.
Image Optimization
Use Next.js <Image> for all static assets (CV preview, Project screenshots).
All images must have alt text for accessibility.
Use WebP formats where possible.
Layout Optimization
Hero Section: 100vh (viewport height) with the Sun/Menu centered.
Content Flow: Single-column layout for text-heavy sections (History, CV) with max-width containers (max-w-4xl) for readability.
Grid: Use CSS Grid for the "Projects" section (1 col mobile, 2 col desktop).
Component Structure
Atomic Design: Create base components (<Button>, <Card>, <Tag>) that share the "Space" aesthetic (glow effects, rounded borders).
Server Components: Fetch data (Projects, Experience) in Server Components and pass data to Client Components for rendering.

Prompt Part 2: Components Architecture
Components Architecture
1. GalaxyBackground Component (The Core Engine)
Purpose: Render the immersive, interactive 3D nebula that persists across the entire application. This is the visual foundation.
Requirements:
Technology: React Three Fiber (<Canvas>) + Custom Shaders.
Visual Style:
Diagonal Flow: A cosmic river flowing from Top-Left to Bottom-Right.
Palette: Deep Violet (#1a0b2e), Magenta (#ff00ff), Cyan (#00ffff), and Deep Space Blue (#050510).
Core: A bright, white-cyan nucleus slightly offset from the center.
Performance: Use InstancedMesh for stars (count: ~5000) to ensure 60FPS on mobile.
Key Features:
Interactive Parallax: The galaxy rotates slowly by default. When the mouse moves, the rotation speed and camera angle shift subtly (parallax effect).
State Awareness: Must listen to a global store (Zustand).
State 'Passive': Slow, majestic flow.
State 'Active' (Menu Open): The galaxy accelerates slightly or "warps" backwards to create depth.
Star Layers:
Layer 1: Distant dust (fog).
Layer 2: Small white stars.
Layer 3: Rare, bright "Hero Stars" with 4-point diffraction spikes (using sprites).
2. SunNavigation Component (The Central Hub)
Purpose: Replaces the traditional navbar. A central 3D Star/Sun that acts as the interactive menu trigger.
Requirements:
Position: Fixed at the center of the viewport (z-index: 50).
Visuals: A glowing sphere using MeshStandardMaterial with an Emissive glow (Orange/Gold to contrast with the Violet Galaxy).
Interaction (The "Burger" Replacement):
Hover State:
The Sun scales up (1.2x).
A ring of "Satellite Stars" (Menu Items) orbits into view.
Sound effect: Subtle low-frequency hum (optional).
Click/Active State:
The Sun moves to the top-center or becomes a background element.
Triggers smooth scroll to specific sections.
Sub-Components (The Satellites):
Create floating HTML buttons (using @react-three/drei <Html>) that appear around the sun on hover.
Labels: "My Journey", "Projects", "CV", "Contact".
Animation: They should spiral out from behind the sun when hovered.
3. TimelineSection Component (Journey & Experience)
Purpose: Display "Collège/Lycée", "Higher Education", and "Professional Experience" in a vertical, connected layout.
Requirements:
Data Source: Receives an array of Experience objects (Date, Title, Description, Type) via props.
Layout:
Desktop: Alternating layout (Left/Right) with a central "Constellation Line" connecting them.
Mobile: Single column on the left, line on the left.
Visual Style:
Cards: Glassmorphism style (backdrop-filter: blur(10px), bg-white/5, border-white/10).
Typography: Clean sans-serif. Dates in "Accent Cyan".
Connectors: The vertical line should look like a laser or stardust trail.
Animation Specifications:
Scroll Trigger: As the user scrolls down, the "Constellation Line" draws itself (SVG path animation).

Prompt Add-on: Advanced Visual Fidelity & Polish
CRITICAL DESIGN INSTRUCTION:
The visual output must not look like a standard HTML website. It must feel like a cinematic interface. To achieve this, you must implement the following Post-Processing and Micro-Interaction rules.
1. Post-Processing Pipeline (The "Wow" Factor)
Use @react-three/postprocessing to enhance the R3F Canvas. The galaxy and sun must not look flat.
Bloom Effect: Apply a selective Bloom (Threshold: 1.5, Intensity: 1.2) specifically to the Sun core, the "Hero Stars," and the Cyan nebula filaments. They must appear to emit real light that bleeds into the surroundings.
Noise/Grain: Add a subtle Noise effect (Opacity: 0.02) over the entire screen to prevent color banding in the dark gradients and give a "film" aesthetic.
Vignette: Darken the corners of the screen significantly to focus attention on the center (The Sun/Content).
2. Shader Quality (The Nebula)
Do not use a simple static image. The Nebula must be generative or use high-resolution layered textures.
Movement: The fog must flow organically (using Simplex Noise in the shader).
Depth: Use additive blending for the colors. When a pink layer overlaps a cyan layer, it should create white/bright intersections, mimicking real light physics.
3. "Frosted Glass" UI Realism
The UI cards (History, Projects) must look like physical glass floating in space.
Backdrop Filter: backdrop-blur-xl (Tailwind).
Border: The border must be a linear gradient (Top-Left White to Bottom-Right Transparent) to simulate a light source hitting the glass edge.
Shadows: Do not use black shadows. Use colored shadows (e.g., a faint purple glow behind the cards: shadow-[0_0_30px_-10px_rgba(112,0,255,0.3)]).
4. Micro-Interactions & Physics
Magnetic Buttons: The "Sun Menu" items and main CTA buttons should have a slight "magnetic" effect (move slightly towards the cursor when hovered).
Smooth Scrolling: Implement Lenis or a similar library for momentum-based scrolling. The standard browser scroll is too choppy for a space theme.
Text Reveal: Text should not just "appear." It should split by words or characters and fade in with a slight stagger effect (staggerChildren in Framer Motion).
5. Mobile Considerations
On mobile, disable the heavy Post-Processing (Bloom) to maintain performance, but keep the Glassmorphism and Typography hierarchy intact.
